Why On-Device?

The compliance risk of the alternative isn't hypothetical. It's structural.

The cloud interpretability gap: AWS SageMaker Clarify, Google Vertex Explainable AI, and self-hosted SHAP servers all require model inputs to be transmitted to a remote endpoint. For deployments where those inputs contain Protected Health Information or payment-linked features, every explanation request is a regulated data transmission requiring a BAA (HIPAA) or lawful basis assessment (GDPR). Most Android deployments lack this coverage — not by choice, but because the interpretability vendor relationship was never formally scoped.

🔒

Data Residency

Model inputs never leave the device. No BAA required for the interpretability tier. No GDPR transfer mechanism needed. The data pathway simply doesn't exist.

⚡

Zero Latency

Attribution runs as a structured coroutine in the same process as inference. Median 22ms on flagship, 38ms on mid-range. Remote SHAP adds 180ms median under ideal conditions — and fails completely offline.

📡

Offline Reliability

Clinical tablets in rural clinics. POS terminals in connectivity-constrained environments. Logistics scanners in warehouse dead zones. Interpretability works regardless of network state.

📋

Audit-Ready

Every explanation artifact — attribution vector, confidence interval, input hash, model version — is written atomically to an AES-256-GCM encrypted local audit log for regulatory inspection.

Quick Start

Five lines. One suspend function. Zero network calls.

Kotlin build.gradle.kts 
implementation("com.clearcode:ai-interpretability-android:1.0.0")
Kotlin InferenceViewModel.kt 
val interpreter = InterpretabilityInterpreter.wrap(tfliteInterpreter, config)

// In your ViewModel coroutine scope:
val result: AttributionResult = interpreter.runWithAttribution(inputTensor)
val topFeatures = result.attributions.sortedByDescending { it.magnitude }.take(5)
AuditLogger.record(result, context)

Works with TFLite, MediaPipe Tasks, and ONNX Runtime. See the integration guides for full setup.
